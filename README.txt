Добро пожаловать в Omega.Downloader!
Здесь вам расскажут и покажут(нет) что, зачем и почему.
1. Для работы с артифактори необходим менеджер пакетов conan. 
 1.1 Если conan есть и настроен:
	a. Узнать имя automiq/build репозитория через команду в cmd "conan remote list".
	b. Перейти в Omega.Downloader пункт меню File->Settings, в поле conan repository ввести имя, а так же выбрать путь для папки-загрузки дистрибов (не требующую доп. прав).
	c. Через окно логина, ввести доменные имя и пароль. 
			Ремарка: если, при попытке загрузить список версий, появляется окно с ошибкой логина. Можно залогиниться через cmd командой "conan user <имя> -p <пароль>"
	d. Загрузить список версий, нажав на интересующий из проектов.
	e. Открыть интересующую версию. Открывать можно несколько окон, так же при переключении проектов, окна остаются действующими.
	f. Добавить билды в очередь на скачку.
	g. Нажать кнопку download. Вся информация о скачке отображается в Logger-окне.
 1.2 Если conan отсутствует или не настроен, молимся всем богам что хотя бы pip настроен.
		Устанавливается в cmd "pip install conan". Если pip не найден, то качаем распоследний питон и устанавливаем pip.
	a. Убедиться что версия conan свежая(удалить старую и поставить новую, если уже был)
	b. Проверить, что cmd распознает команды conan.(Ввести "conan -h")
	c. Перейти в папку conan(папка у пользователя .conan) и открыть remotes.json. (Если его нет, ввести в cmd "conan remote list" и файл создастся)
	d. Добавить нижестоящую json структуру в добавок к дефолтной
	,{
		"name": "Automiq",
		"url": "https://artifactory.automiq.net/artifactory/api/conan/automiq-build/",
		"verify_ssl": true
	}
	e. Через Omega.Downloader или cmd произвести логин для добавленного репозитория(conan user <имя юзера> -p <пароль> -r <имя репозитория>)
	f. Проделать все что описано в п 1.1.d.

2.	Доп возможности:
	2.1 Имеется возможность сохранять добавленные в очередь ссылки в файл. Для этого в меню: Options->Save queue. Файл сохраниться в папке загрузки, указанной в настройках, так же оповестит в logger-окне.
	2.2 Можно загрузить файл со ссылками в очередь Options->Load queue.
	2.3 Можно посмотреть установленные компоненты alpha платформы. Options->Installed components.
	  2.3.1 Так же, выбрав необходимые компоненты, можно их удалить.
	  2.3.2 Выбрав компоненты кроме: виндовых Алармов, Трендов, агента лицензирования, Поинта, серверов отчета и тулзов, нажать save. Все выделенные компоненты сохранятся в файле в виде ссылок для передачи/добавления в очередь на загрузку. Сообщения о сохранении будет в Logger-окне.
	2.4 По кнокпе меню Storage октрывается проводник с папкой загрузки файлов.
	2.5 Если Omega.Downloader запущен, то свернуть/развернуть его можно по "ctrl+alt"(не настраивается).
	
3. Особенности работы:
	3.1 Можно добавлять произвольное количество проектов. Даже других брендов, например Astra. Не рекомендуется нажав add project и не вводя данные закрывать окно настроек. Все настрйоки сохраняются при нажати ok. ( так получилось:) )
	3.2 Можно скачивать дистрибутивы проектов в отдельные папки, для этого в настройках установить enable custom path и отредактировать при необходимости поле custom path. Так же можно скачивать только виндовые дистрибы или линуксовые или вместе.
	3.3 Можно удалить добавленные проекты. Нажав на крестик. Удаление безвозвратно.
	3.4 Количество окон с билдами версий неограниченно. Даже при загрузке другого проекта, они продолжат работать. Можно таким способом кешировать версии для ускорения работы.
	3.5 Если не работает загрузка пакетов на каком либо этапе, рекомендуется проверять conan чере cmd. Скорее всего с ним возникли трудности.
	3.6 Логин обязательно проводить с паролем, так как артифактори защищен.
	3.7 Проверялось все на винде и пока что заточенно только под нее. В будущем может быть добавлена поддержка и на линуксе.

Во славу автоматизации и удобизации(надеюсь)!
